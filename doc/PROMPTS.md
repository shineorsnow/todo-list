# 待办清单项目 - 开发提示词记录

> 本文档记录项目开发过程中使用的提示词，便于后续复用和版本追溯。

---

## 版本：v1.0.0

### 提示词 1：需求分析与计划

```
我想做一个待办清单网页，要求：
1. 可以添加新任务（输入框+添加按钮）
2. 可以点击任务标记为完成（会有划线效果）
3. 可以删除任务（每个任务旁边有删除按钮）
4. 界面要简洁美观，用现代化的设计

请先给我一个实现计划，包括用什么技术、分几个文件、大概的实现步骤，等我确认后再开始写代码。
```

### 提示词 2：文档与版本控制

```
将过程中对应的提示词写成md文件，还有一个操作文档和需求环境手册，注意好需要有版本号。
然后现在有什么工具可以实现当前版本和其他版本之间的改变。比如建立git本地仓库。
将这些厘清变成一段提示词后给我，然后使用
```

## 完整开发提示词模板

以下是可以用于类似项目的完整提示词模板：

```
我想开发一个【项目名称】网页应用，要求：

## 功能需求
1. 【功能1描述】
2. 【功能2描述】
3. 【功能3描述】

## 设计要求
- 界面简洁美观，现代化设计风格

## 交付要求
- 请先给出实现计划（技术选型、文件结构、实现步骤）
- 确认后再开始编码
- 需要创建以下文档：
  - README.md（项目说明、环境要求、操作文档）
  - PROMPTS.md（开发提示词记录）
  - 版本号管理（使用 Git 标签）
```

---

## 版本管理提示词

用于后续版本迭代时使用：

```
当前项目版本为 v1.0.0，现在需要更新以下功能：

## 新增/修改需求
1. 【新功能描述】

请完成以下工作：
1. 实现新功能
2. 更新 README.md 文档
3. 更新 PROMPTS.md 记录本次提示词
4. 使用 Git 提交并打上新版本标签 v1.x.0
```

## 版本差异查看

```bash
# 查看两个版本之间的代码变化
git diff v1.0.0 v1.1.0

# 查看某版本的详细改动
git show v1.1.0

# 查看版本历史
git log --oneline --decorate
